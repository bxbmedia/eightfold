Project Overview

Fork in the Road is a landing page for a fictional food truck business. It was built as part of a Claude-assisted development exercise to explore how AI tools can accelerate frontend development while keeping the developer in full creative control.

The site features a hero section with parallax scrolling, a menu displayed with interactive card flip animations, a weekly schedule calendar, and a signup form with a confetti celebration on submit. The entire project uses vanilla HTML, CSS, and JavaScript with zero dependencies ‚Äî no frameworks, no build tools, no npm install.

AI Context (claude.md)

The claude.md file gave Claude the project scope, design constraints, and creative direction upfront. The key decision was specifying vanilla-only ‚Äî no frameworks. This kept Claude focused on clean, readable code instead of reaching for React or Tailwind. The food truck theme and interactive elements (card flips, confetti, parallax) were defined as requirements so the AI had clear targets to hit.

claude.md 
# Fork in the Road ‚Äî Food Truck Landing Page

## Project Context
Build a single-page landing page for a fictional food truck.
Vanilla HTML, CSS, and JavaScript only ‚Äî no frameworks, no dependencies.

## Design Direction
- Dark theme with warm accent colors (food/street vibe)
- Interactive elements: card flip menu, parallax hero, confetti on signup
- Mobile-responsive, smooth scroll animations
- Mouse glow effect for visual polish

## Constraints
- Zero npm dependencies
- Single index.html entry point
- All styles in dedicated CSS file
- All interactivity in dedicated JS file

Reflection

The biggest takeaway was how much the claude.md file shapes the output. When the constraints are clear and the creative direction is specific, the AI stays on track and produces code you actually want to keep. When the context is vague, you spend more time correcting than building.

Going vanilla was the right call for this exercise. It forced both me and the AI to work with fundamentals ‚Äî no hiding behind framework abstractions. Every animation, every interaction had to be written from scratch. The result is a lightweight page that loads instantly and has zero build complexity.
